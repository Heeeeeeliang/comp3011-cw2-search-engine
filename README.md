# COMP3011 Coursework 2 – Search Engine Tool

A command-line search engine that crawls [quotes.toscrape.com](https://quotes.toscrape.com/), builds an inverted index of every word on the site, and answers single- and multi-word queries against it.

## Project structure

```
src/
    crawler.py      # politely crawls the target site
    indexer.py      # builds and persists the inverted index
    search.py       # query logic (print, find)
    main.py         # interactive command-line shell
tests/
    test_crawler.py
    test_indexer.py
    test_search.py
data/
    index.json      # generated by the `build` command
requirements.txt
README.md
```

## Setup

```bash
python -m venv .venv
source .venv/bin/activate           # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

## Running

From the project root:

```bash
python -m src.main
```

You will see a `>` prompt. The available commands are:

| Command            | Description                                                              |
| ------------------ | ------------------------------------------------------------------------ |
| `build`            | Crawl the site, build the inverted index, and save it to `data/index.json`. |
| `load`             | Load the previously-saved index from disk.                               |
| `print <word>`     | Print the inverted-index entry for `<word>` (frequency + positions per URL). |
| `find <w1> <w2> …` | List every page containing **all** of the given words (AND query).       |
| `exit`             | Quit the shell.                                                          |

### Example session

```
> build
Crawling https://quotes.toscrape.com/ ...
Indexed 10 pages, 1234 unique words. Saved to data/index.json.

> find good friends
http://quotes.toscrape.com/page/3/

> print nonsense
{
    "http://quotes.toscrape.com/page/5/": {"freq": 1, "positions": [42]}
}
```

## Design notes

- **Politeness window.** The crawler sleeps at least 6 seconds between successive HTTP requests, as required by the assignment brief.
- **Inverted index structure.** A nested dictionary `word -> url -> {freq, positions}`. JSON is used for persistence so the index file is human-readable and easy to inspect.
- **Case-insensitive search.** All tokens are lower-cased at index time and at query time.
- **Tokenisation.** Words are extracted by the regex `[a-z0-9']+` after lower-casing, which keeps contractions like `don't` together.
